Why python sucks (in my opinion)

First of all, let's say: I hate python. Really much. Like, a lot. Very deeply too. Most because I used it in my early programming skills' development process and I know it relatively well. And I'm writing this article because my friend asked me to most because I think python is a god-like language, a bit like C, but in contrast, C don't have much issues intrinsic to it, but most because of its maintainers (and the ABI problems generated by different maintainers). Python issues, in the other hand, are must because of it itself, it's design, it's implementation and developers (once there is not, like C, dozens of main python interpreters). So here we go:

Python is strange

Python, and other languages like Ruby, has this weird paradigm of "readability" that creates really irritating things. First of all, the indentation; something so relative and taste dependent should not be so important in the implementation of the code. If you already have written a python code in the default IDE for windows and then openned the file in another code editor and changed it, you know what is pain. If, for some reason, you wrote code using hard tabs instead of spaces, you just have almost completely fucked your program. Of course, you can (not that) easely revert it, but still is really bothering have to do it.

Second: it's partially OOP. I make the Brian's words mine. And I also add some, making every variable an object is needless and sometimes expenssive. And I didn't even start talking about performance. All number and string instances having tons of functions attached to them, and copied every assignment, is really expenssive (an int is 24 bytes!)

And third: the definition of variables. What the fuck who things that system is useful? You may end overwriting your outer-scoped variables by simply assigning something to it again. An assignment in python is indistinguishable of an namespace definition. And that really sucks. The fact that you need to use global foo to access variables of outer scopes is really strange. And most tutorials don't tell you (at least Sololearn didn't to me) that you can use it. This "feature" can lead to very invisible bugs that may destroy your database or something (first of all, you shouldn't be using python).

Python is slow

Python is interpreted, what is not bad by itself, but can lead to a lot of problems. Some like bad habits to new programmers (python is a really bad first language, by the way). Dynamic typing may sound cool and easy but it leads to tons of problems of runtime assertions or failures, and leads to slow runtime too. And python is reeeealy slow. Lua is my jam, I know it have some problems (like being also dynamic typed, only had implemented constants recently and so on), but it at least is really fast. See this site to check it out. Lua is sometimes ~5 times faster and ~7 times cheaper. That's a lot. Holy crab, that's a lot. Why people still use it? I tell you: it's easy to learn (marromeno tamb√©m) and easy to start using. It have some vicious things like garbage collector and exception handling (that by the way Ginger Bill already wrote a really good article on why exceptions are bad). And it leads us to the next topic.

Python is useless

"But hey! I use python to big numeric calculations with NumPy and do deep learning things with this another module that I have to swap across versions and bla bla bla", no. Most "really useful modules" of python are written in C/C++ because python can't handle this. And if you are using a toy language to use this modules, I thing it's much more simpler and productive simply use this modules directly, aka don't use python. If you need another language to make your own useful, then your language is not useful. If python depends on code of other languages to don't become useless, then it is already useless. It's simple. If I need a rocket to make my car fly to the moon, it's much better simply use the rocket and get rid of the car entirely.

Python is messy

You may already know that there is 2 types of python programmers: the really strange and antiquate masochists that use python2 and the weirdos that use python 3. I what universe a language developer (and by it understand organization or people behind it) would create an language A, then after some years create language B, that is kinda like A but is almost entirely incompatible with it, and say "hey, this old language that you all use will soon be terminated and shut down, please migrate all your code base to this new one". That's horribly cruel and yet comical. I think you all deserved this, actually. The first mistake was use Python. And now, if you want to install some module to your python 2, you need to use pip2 install foo_bar. But if you are using python3 everything will be just fine, right? WRONG! A lot of modules are not retro-compatible. or forward-compatible. So many times I had really crazy errors of dependency while trying to install pyxel in my machine. "Oh, your module A needs B, but we can't install it because python3.6 is using B 1.3, and you now need B 2.7, but B 2.7 is not supported in python3.1 that you are using now". Then you try to install it using the pip of python3.6, but for some reason your system considers pip as the python3.9's pip. And then the Dependecy Hell of C becomes a new Hell in Python. You really like python, dude? My freaking god.

The end

Python is one of the worst programming languages. End. That's it. There is no "I know that many programmers like and use python" or "python has its place and room to grow". Python is a cancer that destroys everything I love. Please don't use this piece of shit.
